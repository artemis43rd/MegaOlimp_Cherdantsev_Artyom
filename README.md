# Football Stats App

Мобильное приложение на React Native для отслеживания футбольной статистики, результатов матчей и анализа игр. Приложение интегрировано с API SStats.net и обладает продвинутой системой кэширования и обработки сетевых ошибок.

## Реализованный функционал

### 1. Авторизация
* Реализован универсальный экран входа/регистрации.
* Система настроена на имитацию успешного входа: любые введенные учетные данные позволяют получить доступ к приложению.

### 2. Главный экран (Home)
* **Список матчей:** отображение предстоящих игр на ближайшие 2 дня.
* **Информативность:** страна проведения, лига, время, названия команд.
* **Умные метки:** автоматическая замена даты на статус "Today" или "Tomorrow" для актуальных игр.
* **Пагинация:** кнопка "Load More" для ручной подгрузки дополнительных матчей из текущей выборки.
* **Интерактивность:** Название лиги на карточке матча кликабельно — нажатие на него автоматически переводит пользователя на экран поиска с уже выбранной лигой.

### 3. Обработка сетевых неполадок и Кэширование (Бонусные баллы)
* **Уведомление о неполадках:** реализован перехват ошибок запросов к API. При возникновении проблем с сетью пользователь видит информационный баннер в верхней части экрана.
* **Логика Кэширования:** при неудачной попытке обновить данные приложение автоматически загружает из `AsyncStorage` последние успешно полученные матчи (до 30 записей). При этом выводится уведомление о том, что данные взяты из кэша.

### 4. Поиск и Фильтрация (Search)
* **Фильтр матчей по командам:** реализовано текстовое поле поиска. Пользователь может найти все предстоящие или завершенные игры конкретного клуба.
* **Временные отрезки:** возможность фильтровать игры на "День", "Неделю" или "Месяц".
* **Архив:** отдельная кнопка для просмотра завершенных матчей в выбранной лиге за указанный период времени.
* **Умный список лиг:**
  * Автоматическое удаление дубликатов из API.
  * Форматирование названий с указанием страны (например, "Cup (Spain)").
  * Алфавитная сортировка для удобства навигации.
  * Пункт "All Leagues" для сброса фильтра по турниру.

### 5. Детализация и Навигация
* **Match Details:** детальный отчет о матче в модальном окне с форматированным временем (например, Friday, February 6, 2026 at 02:30).
* **Быстрый переход:** на карточке матча название лиги является кликабельным — нажатие на него переносит пользователя на экран поиска с уже выбранной этой лигой.

## Требования к окружению (Важно для проверки)

Приложение разработано и протестировано со следующими параметрами. **Пожалуйста, убедитесь, что ваше окружение соответствует им:**

* **Node.js:** v18 или выше.
* **JDK:** Версия **17** (обязательно для корректной сборки Gradle).
* **Android SDK:** API Level **34** (Android 14.0 "UpsideDownCake").
* **Android Build Tools:** версия 34.0.0.
* **Эмулятор:** Рекомендуется Pixel 7 (x86_64) с API 34.

## Инструкция по запуску


Для сборки и запуска приложения требуется **Android Studio** и настроенная среда **React Native**.

### 1. Настройка окружения (Linux)
Перед запуском убедитесь, что у вас настроены переменные окружения для Android SDK. Добавьте следующие строки в ваш `~/.bashrc` или `~/.zshrc`:

```bash
export ANDROID_HOME=$HOME/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools
```
### 2. Клонирование и установка
```bash
git clone https://github.com/artemis43rd/MegaOlimp_Cherdantsev_Artyom.git
cd MegaOlimp_Cherdantsev_Artyom
npm install
```
### 3. Настройка Java (если у вас несколько версий)
Убедитесь, что JAVA_HOME указывает на 17-ю версию.
```bash
java -version
```
### 4. Запуск Metro Bundler
В первом окне терминала запустите сервер сборки:
```bash
npx react-native start
```
### 5. Запуск на устройстве
Во втором окне терминала (при запущенном эмуляторе Android):
```bash
npx react-native run-android
```
